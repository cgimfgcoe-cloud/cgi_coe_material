01 — Introduction
What is Oracle Ask EBS?
Key Benefits for Manufacturing
Oracle Ask EBS Defined
Oracle Ask EBS is an AI-powered natural language interface embedded within Oracle E-Business Suite (EBS). It enables users to query enterprise data, retrieve business insights, and navigate EBS functions using plain English — without requiring SQL knowledge or complex navigation through menus. Powered by Oracle's AI Foundation, Ask EBS interprets user intent and translates queries into EBS data calls across modules including Manufacturing, Finance, Procurement, and Supply Chain.

Get instant answers to production, inventory, and quality questions without waiting for reports or IT support.
📚 Reduced Training

🔗 Cross-Functional

📋 Audit-Ready

10 — Troubleshooting & FAQs (Part 1)
Common Issues & Solutions
⚠ Query returns no resultsLOW
Cause: Over-restricted filters, wrong org code, or no data matching criteria
✅ Remove one filter at a time. Verify org code (e.g. M1 not MFG01). Try broader date range. Check if items exist in selected sub-inventory.
⚠ "I don't understand your question"LOW
Cause: NLP parser couldn't map query to an EBS module or entity
✅ Rephrase using EBS terminology. Include module keyword (e.g. 'work order', 'PO', 'AP invoice'). Use query suggestions from the autocomplete dropdown.
⚠ Slow response / timeoutMED
Cause: Query scope too wide, peak system usage, or large dataset
✅ Add date range filter (e.g. 'last 30 days'). Limit to one org/plant. Run during off-peak hours. Contact EBS Admin if persistent.
⚠ Data doesn't match EBS formHIGH
Cause: Query hitting incorrect module, stale cache, or report logic difference
✅ Verify you're querying the correct org. Clear session and retry. For period-end, use standard Oracle reports as source of truth.
⚠ Access denied / missing moduleMED
Cause: User role doesn't include required responsibility
✅ Submit IT access request for additional EBS responsibility. Reference role matrix in SharePoint COE library. Include manager approval.
⚠ Export fails / blank fileLOW
Cause: Browser popup blocker, session timeout, or large result set
✅ Allow popups for EBS domain. Re-login and retry. Limit result set to <10,000 rows. Use scheduled delivery for large exports.

10 — FAQs (Part 2)
Frequently Asked Questions
Can Ask EBS modify data in Oracle EBS?
No. Ask EBS is read-only. It retrieves and displays data but cannot update, create, or delete any EBS transactions. All write operations must be performed through standard EBS forms or workflows.
How current is the data shown in Ask EBS?
Most transactional data (work orders, inventory, POs) is real-time. Financial summaries and some reports may be refreshed on a scheduled basis (typically T-1). Check the 'Data as of' timestamp in the results header.
Can I use Ask EBS on mobile devices?
Yes. Ask EBS is responsive and works on tablets and smartphones via the EBS mobile URL. Voice input is supported on mobile for hands-free querying on the shop floor.
Is my query history stored? Who can see it?
All queries are logged for 90 days for system performance and audit purposes. EBS Admins and Compliance officers can review query logs. Queries are not visible to other regular users.
What if Ask EBS gives me incorrect data?
Use the 'Report Issue' button in the results panel. The data team will investigate within 1 business day. For critical decisions, always cross-reference with the source EBS form or standard Oracle reports.
Can I embed Ask EBS results in other tools?
Yes. Results can be pushed to Oracle Analytics Cloud (OAC), exported to Excel, or emailed via the distribution feature. Direct API integration with third-party BI tools is available with IT configuration.
How do I request a new query type not currently supported?
Submit an enhancement request via the CGI COE ServiceNow portal (IT category: EBS Ask Enhancement). The COE team reviews requests bi-weekly and prioritizes by business impact.

01 — Introduction
What is Oracle WebADI?
Web Applications Desktop Integrator — Key Benefits for Manufacturing COE
Oracle WebADI Defined
Oracle Web Applications Desktop Integrator (WebADI) is an Oracle EBS framework that allows users to interact with EBS data using Microsoft Excel as the front-end interface. WebADI bridges the gap between the familiar Excel environment and the enterprise EBS system — enabling mass data entry, bulk updates, and structured downloads without requiring technical SQL or API knowledge. Introduced in EBS 11i and enhanced through R12, WebADI is widely used across Finance, Manufacturing, HR, and Supply Chain.
📦 Mass Data Entry
Upload thousands of work orders, routings, or BOM components in one operation — eliminating manual EBS form entry.
📊 Familiar Excel
No new tools to learn. Users work in Excel with full formatting, formulas, and validation — data syncs to EBS on upload.
🔄 Bidirectional Flow
Download EBS master data for review and correction in Excel, then push updates back — keeping data accurate at scale.
✅ Reduced Errors
WebADI LOVs enforce valid EBS lookups directly in Excel cells, preventing invalid data before upload.

07 — Filters, Parameters & Prompts
Making Dashboards Interactive
🎛 Dashboard Filters
Global filters applied to all visuals on a canvas. Set from the filter panel (funnel icon). Users can adjust values at runtime. Filter types: text match, date range, top-N, numeric range.
Example
Plant = 'A01' AND Period BETWEEN 'Jan-26' AND 'Mar-26'
💡 Use 'Apply to All Canvases' to share a filter across all tabs in a workbook.
📊 Visual-Level Filters
Filters applied to a single visualisation only. Right-click the visual → Add Filter. Useful when one chart needs a different scope than others on the same canvas.
Example
Chart showing only 'Scrap > 5%' while KPI tile shows total production
💡 Visual filters override dashboard filters — use carefully to avoid user confusion.
🎨 Conditional Formatting
Colour-code values based on rules. Applied per column in data tables or KPI tiles. Uses measure-based or comparison rules.
OEE Colour Scale
Red <75% · Amber 75–85% · Green >85%
💡 Use the same colour thresholds across ALL OEE visuals for consistency.
⚙️ Parameters & Calculations
Create calculated measures referencing parameters in formulas. Use AGO() for period-over-period comparisons. Store in Subject Area for reusability.
Calculated Measures

PPV = (PO Price − Std Price) × Qty
💡 Store reusable calculations in a Subject Area — not per-workbook.

13 — Best Practices (Part 1)
Design, Performance & Data Modelling
✅ DO — Best Practices
Use certified datasets in the OAC Data Catalog as single source of truth
Apply row-level security at the dataset level — not per-workbook
Limit live EBS queries to <100K rows. Aggregate large datasets in ADW first
Use the OAC Data Flow to pre-calculate OEE, variance, and KPI metrics
Version-control workbooks by publishing to Shared Folders with naming convention
Schedule data refreshes during off-peak hours (11 PM–5 AM)
Use OAC 'Explain' before building charts — let AI surface hidden patterns first
❌ DON'T — Common Mistakes
Let every user build their own dataset from raw tables — creates metric inconsistency
Rely on users 'not knowing' where data is — security by obscurity fails
Run multi-million row JDBC queries directly against production EBS during business hours
Calculate complex KPIs in every individual workbook — creates maintenance nightmare
Save workbooks as 'Test1', 'Final', 'Final_v2' in personal folders
Auto-refresh every minute on a 500K-row EBS dataset — will lock database sessions
Spend hours building charts for insights OAC Auto Insights finds in 30 seconds
💡 CGI COE Tip
Use Oracle ADW as the analytics warehouse between EBS and OAC. Pre-aggregate and join in ADW — keep OAC for visualisation and exploration only.
